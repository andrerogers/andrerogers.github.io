Voxels

What are voxels? A pixel can be understood to be an (x, y) co-ordinate on a 2D plane. A voxel is a pixel whose structure exists in the third dimension as well, i.e. it has depth (z axis). It represents a single value of a given object stored at an (x, y, z) co-ordinate in a 3D data structure. The value itself only specifies the color at that spatial position, its position is deduced by its neighbors and by the structure itself. Voxels can be rendered by either using volumetric data and rendering it with a direct volume rendering technique such as using slices or raycasting. Another alternative is extracting isosurfaces of a model which are computed and often rendered with the marching cubes algorithm.

Voxels if processed and stored optimally can give rise to high LOD's at any or most ranges. Static models are suited well for this mode of rendering, but when these models need to be animated or transformed along complex tree heirachies there is an overhead to re-visualizing the data within the data structure as data needs to be shifted between positions in the structure to render the given animation. Voxel's are being moved. Thus, complexity increases even with algorithms more complex then the mundane naive and linear approaches.

The project described here is based on a direct volume rendering approach via rayvcasting. It began with trying to store the data within an octree, and making and maintaining the sparse nature since all nodes of the octree were not occupied to represent a given volume. Due to the lack of time, the project was diverted to trying to create an interactive way to light the volume, and the effect pursued is a toon effect, the exaggeration of colors manipulated by a light source.


Interactive Volume Rendering

I augmented my implementation of the paper "XToon: Extended Toon Shader" by Pascal Barla and incorporated it into the rendering of voxels (Implementation can be found here) and used it as a transfer function to shade the volume. As described by the paper, a 2D texture is used as a lookup for shading where a computed detail value (D) is understood as the y axis and the lambertian model (N.L) is used as the x axis.

Here, I compute the detail (D) as the density of the volume. A gradient is computed with the sobel operator and is applied per fragment of voxels to threshold, and its magnitude is applied per fragment. These two values are used for the transfer function look up.

The perks of this paper which are depth of field, aerial perspective, specular highlights, and backlighting can be trasferred to rendering volumes. This requires adjusting the shader to work with volumetric data, and designing textures which will help exhibit the desired effect.


Future Work

The code for the project can be found here. There are enumerable things that can be done to improve the project, possibly even correct it. There are some things that I would like to do to it which are as follows,

  • - Further optimizing and maintaining a sparse octree, which can shift data between its nodes at a feasible rate.
  • - To further bolster the profile of an optimized, efficient sparse octree I would like to try and implement a simple ball bouncing while maintaing the structure of the object all through the animation via the octree.
  • - Applying the above mentioned XToon implementation to other facets of volumetric rendering, possibly volumetric light or fog, or volumetric fluid.
