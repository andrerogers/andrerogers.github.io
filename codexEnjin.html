In my free time I enjoy learning and building applications with OpenGL. Whether its writing a shader, or trying to simulate a ball bouncing, water flowing, or a particle explosion. Writing code for these applications really amuses me, and challeneges me at the same time. And I do love writing it in C++, I can't wait for the day when I write C++ code without a single leak. Bug free code, through and through. Its striving for this almost impossible reality is what I feel will make me better at what I like doing.

With that in mind, I want to slowly try and build an engine, which will help me when I want to create one of my hobby applications in OpenGL. I want it to be something that I can reuse and maybe put out there for people to critcize, propogate and use.

Core

I will be using OpenGL of version 3 and up. GLFW is used to create the application. It is a multi platform library for OpenGL. Currently I am using it to create a window, give it an OpenGL context and also receive inputs and events.


Renderer

  • - It is currently a forward renderer.
  • - Models are imported via assimp and OpenGL neatly arranges it with Vertex Attribute Objects (VAO's) and Vertex Buffer Objects (VBO's).
  • - Textures are read via SOIL, and OpenGL assigns it a unit on the GPU.
  • - The scene is lit with lights, namely directional, point and spot lights.
